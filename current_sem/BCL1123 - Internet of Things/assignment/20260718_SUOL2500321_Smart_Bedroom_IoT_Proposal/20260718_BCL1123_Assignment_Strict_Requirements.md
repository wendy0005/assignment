## 2. Mandatory report order

- [ ] Cover Page.
- [ ] Clean, logically structured Table of Contents reflecting the required report sections.
- [ ] Executive Summary: briefly summarize the proposal.
- [ ] Target Environment: introduce the selected area, include clear original photographs, and explain its limitations or problems.
- [ ] Proposed Solution and Illustrations: present hardware selections, system architecture, flowchart, and dashboard/application UI prototype.
- [ ] IoT 4 Layers Explanation: explain the Edge, Connectivity, Cloud, and Application layers in the specific context of the proposal.
- [ ] Summary.
- [ ] References in APA format.
- [ ] Link to the externally shared presentation video.

## 3. Case study and problem identification — 15%

- [ ] Select one or two clearly defined accommodation areas.
- [ ] Take and include clear photographs of the actual area.
- [ ] Identify concrete limitations or problems visible in or relevant to the selected environment.
- [ ] Provide solid justification showing why the problem warrants an IoT solution.

## 4. Proposed solution and hardware — 20%

- [ ] Present a viable IoT solution directly addressing the identified problems.
- [ ] Explore [Wokwi](https://wokwi.com/) and select appropriate available sensors and actuators.
- [ ] If using a custom-made Wokwi sensor, explain and justify it clearly.
- [ ] Include pictures of the selected sensors and actuators in the illustrations.
- [ ] Explain the purpose and role of every selected hardware component.
- [ ] Ensure the hardware choices are technically compatible with the proposed system behaviour.

## 5. System design — 20%

- [ ] Include a clear system architecture diagram showing the complete IoT implementation.
- [ ] Include a professional flowchart showing system logic, decisions, inputs, outputs, and control actions.
- [ ] Label every important component and connection clearly.
- [ ] Make diagrams readable at normal document zoom and suitable for academic submission.
- [ ] Use draw.io for diagrams if practical; the brief highly recommends it.

## 6. Dashboard or application UI prototype — 10%

- [ ] Design an application/dashboard wireframe for monitoring and controlling the IoT system.
- [ ] Show the important readings, alerts, controls, device states, and user actions required by the proposal.
- [ ] Make the interface intuitive and responsive in its intended behaviour.
- [ ] Ensure the screens and interactions correspond to the system flowchart and proposed hardware.
- [ ] Use draw.io for the UI wireframe if practical; the brief highly recommends it.

## 7. IoT four-layer architecture — 15%

- [ ] **Edge layer:** identify the sensors, actuators, controller/device, local processing, and physical interactions.
- [ ] **Connectivity layer:** identify the network and communication technologies that carry data and commands.
- [ ] **Cloud layer:** explain remote data ingestion, storage, processing, rules, analytics, security, or services used by the proposal.
- [ ] **Application layer:** explain the dashboard/application, notifications, user controls, and presentation of information.
- [ ] Describe the components of every layer in detail rather than giving only generic definitions.
- [ ] Explain how data and commands travel between the four layers.

## 8. Uniqueness requirement

- [ ] Identify the special or unique idea introduced by the proposal.
- [ ] Compare it with standard IoT applications or existing solutions available in the market.
- [ ] Explain the practical advantage of the proposed distinction without making unsupported claims.

## 9. Report quality, citations, and formatting — 10%

- [ ] Submit the report in **DOCX** format.
- [ ] Use **Times New Roman, 12-point** font.
- [ ] Use justified paragraph alignment.
- [ ] Use first-line paragraph indentation.
- [ ] Use 1.5 line spacing.
- [ ] Use headings and subheadings consistently.
- [ ] Apply bold, italics, and underline only where appropriate.
- [ ] Use both portrait and landscape orientation where the content requires them, with proper page and section breaks.
- [ ] Use different headers and footers for different sections as required by the brief.
- [ ] Format every picture appropriately and insert a caption.
- [ ] Keep all information accurate, clear, and logically structured.
- [ ] Cite every fact, image, diagram, product detail, or other information obtained from the Internet or a textbook.
- [ ] Use APA in-text citations and an APA reference list.
- [ ] Do **not** use Wikipedia as a reference.
- [ ] Use source material sparingly and follow it with the student’s own critical analysis or evaluation.
